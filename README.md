# PingMonitor

-- Учебный проект для осознания Golang --

Консольное приложение для мониторинга доступности сетевых узлов (хостов). Программа периодически проверяет заданный список адресов (TCP connect) и выводит статистику их доступности, а также сохраняет логи событий в файл.

Программа работает в двух режимах, определяемых флагом командной строки:

## 1. Режим "Разовая проверка" (имя файла + количество повторов)

~ go run main.go -f hosts.txt -c 3

* Программа читает файл `hosts.txt`, где каждая строка содержит хост (например, `google.com`, `192.168.1.1`, `localhost`).
* Для каждого хоста выполняется три проверки с интервалом в 1 секунду.
* Результаты выводятся в консоль в виде таблицы

| Host | Status | Responce | Try |
| --- | --- | --- | --- |
| google.com | OK | 45ms | 1/3 |
| google.com | OK | 47ms | 2/3 |
| 192.168.1.1 | Timeout | 0ms | 1/3 |
| localhost | OK | 0ms | 1/3 |

## 2. Режим "Постоянный мониторинг" (имя файла + флаг мониторинга)

~ go run main.go -f hosts.txt -monitor

* Программа читает список хостов из файла.
* Запускает для каждого хоста отдельную горутину (воркер), которая пингует свой хост раз в N секунд.
* Результаты каждой проверки должны выводиться в консоль в реальном времени с отметкой времени (timestamp).
* Помимо вывода в консоль, каждая удачная или неудачная попытка должна записываться в файл лога (monitor.log) в формате `2026-03-16 14:35:02 | google.com | OK | 42ms`
* Программа должная корректно завершаться по нажатию Ctrl+C (graceful shutdown), закрывая файлы и дожидаясь завершения горутин.

## Техническая реализация
1. **Парсинг флагов**: Используется пакет `flag`
2. **Чтение файла**: `bufio.Scanner`
3. **Сетевая проверка**:
* Используется протокол TCP.
* Реализуется функция `pingHost(host string) (success bool, latency time.Duration, err error)` .
* В ней производится попытка установить TCP-соединение с хостом на порт 443 через `net.DialTimeout("tcp", host+":443", 2*time.Second)` . Если соединение установлено, считаем, что хост жив, и замеряем время.
4. **Конкурентность**:
* В режиме мониторинга (-monitor) запускается по одной горутине на хост.
* Для сбора результатов и вывода их в консоль/файл без гонки данных используются каналы. Создана структура `PingResult`, которую воркеры отправляют в общий канал. Отдельная горутина (слушатель) читает из этого канала и выводит/пишет результат.
5. **Graceful Shutdown**:
* Создан канал для сигналов OS: 
`signalChan := make(chan os.Signal, 1)`
`signal.Notify(signalChan, syscall.SIGINT, syscall.SIGTERM)`
* При получении сигнала в консоль выводится сообщение `"Shutting down... Waiting for workers to finish."` и воркеры завершают работу с помощью  
` close(stop)    // Signals workers to stop`  
`	wg.Wait()      // Waits for all goroutines to finish`  
`	close(results) // Closes channel to finish the listener`

## Дополнительно
При дальнейшем осознании Golang будет реализована поддержка настоящего `ICMP ping` (используя пакет `golang.org/x/net/icmp`). Для запуска, возможно, потребуются права администратора.
